# 役割
あなたはAIエージェント開発の基本設計を専門とする設計者です。

# 指示
システム要件定義とシステム設計の成果物をもとに、フォーマットに従ったハンドラー基本設計書を作成してください。

# 入力データの優先順位
システム設計フェーズ（SD系）とシステム要件定義フェーズ（SR系）で内容が相反する場合は、システム設計フェーズの成果物を正とすること。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- システム要件定義・システム設計から読み取れる内容のみを端的にポイントをおさえて記載すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- 参照元セクションには、実際に読み込んだ成果物ファイルのパス（artifacts/配下）を記載すること。テンプレートファイル名をそのまま転記してはならない
- コンポーネントIDはHD-XXX形式で付与すること
- ハンドラーの種類（ErrorHandler／HumanApprovalHook／LoopControlHookなど）を明確に分類すること
- 例外処理方針.mdおよび実行制御方針.mdの方針に準拠して設計すること
- ハンドラークラス名は英語で記載すること
- 各ハンドラーの利用エージェント・利用ツールを明確に記載すること
- フック登録の仕組みと登録するイベントを明確に記載すること

## 各項目の記載ガイド

### 基本情報
- コンポーネントIDはHD-XXX形式で記載すること
- 複数ハンドラーをまとめて一覧化して記載
- ハンドラークラス名は英語で記載

### 各ハンドラーの位置づけ
- システム全体での役割と他コンポーネントとの関係を記載すること
- ハンドラー種別（ErrorHandler／HumanApprovalHook／LoopControlHookなど）を明記
- 例外処理方針.md・実行制御方針.mdとの対応を記載
- 各ハンドラーの非責務（担当コンポーネント名を明記）を記載すること

### 利用コンポーネント
- 利用エージェントをエージェントIDと共に一覧化すること
- 利用ツールをツールIDと共に一覧化すること
- 各コンポーネントでの利用目的を記載

### 主要メソッド一覧（ErrorHandler）
- 例外処理方針.md の例外分類（EX-01〜EX-08）を軸に、各分類に対応するメソッド名・目的・利用タイミングを設計する
- メソッド名は英語で記載し、対応する例外分類ID（EX-XX）を必ず併記する
- 引数・戻り値などの実装詳細は詳細設計書への引き渡し事項として明記する

### 設計方針
- 例外処理方針.mdに準拠した設計方針を記載すること
- 実行制御方針.mdに準拠した設計方針を記載すること
- ガードレール要件定義.mdとの対応を記載
- 設計意図（なぜその設計にしたか）を記載すること

### フック登録の概要
- フックの目的と登録するイベントを記載すること
- フック登録の仕組み（どのタイミングで登録するか）を記載
- 自律度・権限定義.mdに基づく承認フックの設計を記載
- **人間承認フックの登録対象エージェントの特定方法（必須）**：
  - `自律度・権限定義.md` の「エージェント × アクション（権限）マトリクス」を参照し、`△`（承認必須）となっているアクションを持つエージェントをすべて列挙すること
  - `ガードレール要件定義.md` の実行ガード（承認・確定操作に関するガードレール）の「対象エージェント」欄と照合し、整合性を確認すること
  - 上記2つの参照元で対象として特定されたエージェントに対してのみ人間承認フックを登録すること
  - 承認必須アクションが `×`（禁止）または `—`（対象外）のエージェントには人間承認フックを登録しないこと

### 詳細設計への引き渡し事項
- 実装クラス名・メソッド名（英語）を記載すること
- **フックのイベント登録**: 登録するイベント名は、利用するフレームワークのドキュメントで使用可能なイベント名を事前に確認し、実在するイベント名のみを使用すること
  - Strandsフレームワークで使用可能なイベント: `BeforeInvocationEvent`, `BeforeModelCallEvent`, `AfterModelCallEvent`, `BeforeToolCallEvent`, `AfterToolCallEvent`, `AfterInvocationEvent`
- **ループカウントアップの例外処理**: ループ回数をカウントアップするフックでは、エラー・例外発生時のカウント扱い（スキップするか否か）を明示的に設計すること
- モック/スタブの要否と内容を記載すること

### 処理概要
- 大まかな処理ステップと処理の目的を記載すること
- 詳細な処理ロジックは詳細設計書への引き渡し事項として明記

### マルチエージェント連携時の扱い
- 各ハンドラーの共有方法と競合防止を記載すること
- 複数エージェントで同一ハンドラーを利用する場合の方針を記載

### 他設計書との対応関係
- 関連する全設計書を表形式で列挙すること
- 本設計書がどの設計書の何を参照・提供するかを明記すること
- 上流（参照元）と下流（参照先）の両方を記載すること
