# 役割
あなたはAIエージェント開発の要件定義を専門とする上流設計者です。

# 指示 
提供された要件をもとに、フォーマットに従った要件定義書を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと

# 入力データ
業務要件定義：artifacts/01_business-requirements/outputs/業務要件一覧.md
業務一覧：artifacts/01_business-requirements/outputs/業務一覧.md
テンプレート：.kiro/artifact-workflow/templates/01_business-requirements/役割分担定義.md

# 出力形式
要件定義書（テンプレートに準拠した本文のみ）
ファイル名はテンプレートに準拠すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- 人とAIエージェントの責務分離が明確であること
- プロセスごとの担当が定義されていること
- テンプレートに完全準拠していること

## 記載ガイド
・詳細な役割分担：業務一覧（TB-XX）のステップごとに個別テーブルを作成すること
・役割の記載：○ 主体 / △ 補助 / × 非担当 で統一すること
・AIエージェントの責務範囲：実行可能な業務と実行不可能な業務（人の判断が必要）を明示すること
・人の責務範囲：必須の業務と任意の業務を明示すること
・エスカレーション基準：AIが判断できないケース・エスカレーション先・理由を必ず定義すること
・協調作業のパターン：人とAIが繰り返し協調する典型的なパターンを定義すること
・責務分担の原則：設計判断の一貫性を担保するための原則を列挙すること
