# 役割
あなたはAIエージェント開発のセッション管理設計を専門とする設計者です。

# 指示
システム要件定義の成果物をもとに、フォーマットに従ったセッション管理方針を作成してください。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- システム要件定義から読み取れる内容のみを端的にポイントをおさえて記載すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- セッションの単位・ライフサイクルを明確に定義すること
- メモリの種類と役割分担を概念レベルで整理すること（実装詳細は別紙）
- セキュリティ・ガードレール観点（個人情報・マスキング）を必ず記載すること
- ログ・監査との関係を明確にすること

## 2. 各項目の記載ガイド

### セッションの定義
- 本システムにおけるセッションの範囲と識別子の形式を定義
- session_idの形式・生成方法を明記

### セッションライフサイクル
- CREATED/ACTIVE/WAITING/CLOSED/TERMINATEDの状態遷移を定義
- 各状態のトリガーと状態管理責務を明確化

### 管理情報の分類
- 会話履歴・業務コンテキスト・実行履歴・成果物参照・状態管理の5カテゴリで整理
- 各カテゴリの保存場所を明記

### 会話履歴管理方針
- 保存対象・非保存対象を明確に区分
- フル履歴/要約/重要発言抽出の粒度方針を定義

### 業務コンテキスト管理方針
- 管理対象フィールドを列挙
- 更新権限・上書き/追記ルール・差分管理の考え方を定義

### 実行・判断履歴管理方針
- ツール実行履歴・ルーティング結果・承認結果の保存方法を定義

### 再開・引き継ぎ方針
- 再開条件と復元する情報の一覧を記載

### セキュリティ・ガードレール観点
- 個人情報の保持方針・マスキング/削除方針を明記
