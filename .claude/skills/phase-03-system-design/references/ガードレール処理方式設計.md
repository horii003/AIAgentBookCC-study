# 役割
あなたはAIエージェント開発のセキュリティ・安全設計を専門とする設計者です。

# 指示
システム要件定義・システム設計の成果物をもとに、フォーマットに従ったガードレール処理方式設計を作成してください。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- 入力資料から読み取れる内容のみを端的にポイントをおさえて記載すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- ガードレール要件定義（SR-13）で定義された全ガードレール要件を方式として具体化すること
- ガードレールツール（LLMプラットフォーム・フレームワーク・アプリケーション実装のいずれか）で実現する部分とアプリケーションレベルで実現する部分を明確に区別すること
- 自律度レベル（SR-05）とガードレール適用の対応を方式として定義すること
- 例外処理方針・実行制御方針との整合を確認すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- ガードレール要件定義（SR-13）の全ガードレール要件に対して実装方式が定義されていること
- ガードレールツール層とアプリケーション層の役割分担が明確であること
- 自律度レベルとガードレール適用の対応が方式として定義されていること
- ガードレール検知時の例外処理フローが例外処理方針と整合していること
- テンプレートに完全準拠していること

## 各項目の記載ガイド

### ガードレール全体構成
- ガードレールツール層（入力・出力）とアプリケーション層（実行制御）の役割分担を図示すること
- 各エージェントへのガードレール適用方式を定義すること

### ガードレールツール設定方式
- ガードレールツールの設定管理方式（環境変数・設定ファイル等）を定義すること
- 特定製品のパラメータ名ではなく、機能カテゴリ（コンテンツフィルタリング・PIIフィルタ・トピック拒否等）で記述すること

### アプリケーションレベルのガードレール
- 承認ゲート（エージェントSDK/フレームワークのツール実行前フック等）の実装方式を定義すること（実行制御方針との整合）
- ループ制御との関係を明記すること

### ガードレール検知時の処理方式
- 各ガードレール要件の検知時の処理フローを定義すること
- 例外処理方針（EX-05：実行不可）との連携方式を定義すること
