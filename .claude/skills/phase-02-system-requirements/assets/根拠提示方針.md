## 2. 根拠提示レベル定義（プロジェクト標準）

| レベル | 名称 | 提示内容 | 主な用途 | 主体 |
|---:|---|---|---|---|
| 0 | 提示なし | 根拠は内部ログのみ | UX優先・低リスク | |
| 1 | 要約根拠 | 判断理由を自然文で要約 | 一般業務向け | |
| 2 | 参照元提示 | 文書名・規程名・ルール名 | 業務判断の妥当性確認 | |
| 3 | 詳細根拠 | 章番号・条件・数値 | 重要判断・承認前 | |
| 4 | 完全根拠 | 引用全文・適用ルール・入力値 | 監査・法令対応 | |

※ 本プロジェクトで使用する最大レベル：**Lv__**

## 6. 根拠提示フォーマット定義

### 6.1 成果物内での表示位置
- [ ] 本文内
- [ ] 別セクション（「判断根拠」）
- [ ] 折りたたみ表示
- [ ] 別画面／別ファイル

---

### 6.2 表示フォーマット

#### レベル1：要約根拠（例）
```text
本判断は、社内規程および入力された条件に基づいて行われています。
```

#### レベル2：参照元提示（例）
```text
【根拠】
- 適用規程：社内旅費規程 第3条
- 適用ルール：BRL-03（交通費上限判定）
```

#### レベル3：詳細根拠（例）
```text
【判断根拠】
- 適用規程：社内旅費規程 第3条 第2項
- 判断条件：申請金額（15,000円）> 上限（10,000円）
- 適用ルール：BRL-03
- 判断結果：上長承認が必要
```

#### レベル4：完全根拠（例）
```text
【完全根拠】
- 適用規程：社内旅費規程 第3条 第2項（引用：「...」）
- 入力値：申請金額=15,000円、申請者区分=一般社員
- 適用ルール：BRL-03（条件：金額 > 10,000 AND 区分 = 一般社員）
- 中間判断：上限超過 → 承認フロー起動
- 最終判断：承認待ち状態に遷移
```
