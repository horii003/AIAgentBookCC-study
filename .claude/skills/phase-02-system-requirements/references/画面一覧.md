# 指示
業務要件定義とエージェント一覧をもとに、フォーマットに従った画面一覧を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
画面IDは後続工程で不変の識別子として管理すること（SCR-xxx）
CLIベースのシステムの場合は「画面」を「表示・入力インタフェース」として解釈すること
利用者ロール・主エージェントを必ず明示すること
成果物IDとの対応関係を明確にすること

## 2. 各項目の記載ガイド

### 画面IDの採番ルール
- 形式例：`SCR-001`
- 後続工程で不変の識別子

### 画面種別
- 一覧／詳細／入力／確認／ダイアログ／通知 など
- CLIの場合：プロンプト表示／入力受付／結果表示／確認画面 など

### 画面分類
- 業務画面（入力・確認・申請）
- ナビゲーション画面（案内・選択）
- 生成結果画面（文書・データ表示）
- 承認画面（確認・承認・差戻し）
- 例外・エラー画面

### 主エージェント
- 画面上の処理・判断を主導するエージェント（AG-xxx）

### 成果物ID
- 画面が生成・表示する最終/中間成果物（OUT-xxx）

### 画面項目定義
- 入力項目は必須/任意・バリデーション条件を明示する
- 表示制御（権限・自律度・状態による表示切替）を定義する

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- すべての画面にSCR-xxx形式のIDが付与されていること
- 利用者ロールと主エージェントが明示されていること
- 成果物IDとの対応関係が明確であること
- 入力項目の必須/任意・バリデーション条件が定義されていること
- 業務画面/ナビゲーション画面/生成結果画面/承認画面/例外・エラー画面のいずれかに分類されていること
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
