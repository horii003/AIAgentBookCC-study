# 役割
あなたはAIエージェント開発のシステム要件定義を専門とする設計者です。

# 指示
業務要件定義と機能要件一覧をもとに、フォーマットに従ったデータベース以外のデータ一覧を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
データIDは後続工程で不変の識別子として管理すること（DATA-xxx）
データベース（RDB/NoSQL）に格納するデータはテーブル一覧で管理し、本成果物には含めないこと
RAG・ナレッジ・ファイル・ログ・セッション等を対象とすること
機密区分・マスキング要件を必ず定義すること

## 2. 各項目の記載ガイド

### データIDの採番ルール
- 形式例：`DATA-001`
- 後続工程で不変の識別子

### データ種別（分類例）
- ドキュメント（規程、手順書、FAQ、議事録）
- ファイル（CSV、Excel、添付、テンプレート）
- ナレッジ/RAG（ベクトル化対象、検索インデックス）
- メッセージ（メール、チャット、チケット）
- ログ（監査ログ、操作ログ、推論ログ、ツール実行ログ）
- イベント（Pub/Sub、キュー、Webhook）
- 外部SaaSデータ（申請、会計、人事）
- キャッシュ/セッション（会話状態、短期メモリ）
- モデル/学習データ（評価データセット、プロンプト、few-shot例）

### 機密区分
- 公開：制限なし
- 社内：認証必須
- 機微：原則禁止/厳格条件
- 個人情報：原則禁止/最小化

### 作成主体・参照主体の凡例
- 人 / Agent / バッチ / 外部 のいずれかを記載する

### AIエージェント向け追加項目（RAG対象データ）
- 検索対象範囲・チャンク方針・メタデータ・根拠提示方法を定義すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- すべてのデータにDATA-xxx形式のIDが付与されていること
- ドキュメント・ファイル・ナレッジ/RAG・ログ・キャッシュ/セッションのいずれかに分類されていること
- すべてのデータに機密区分（公開/社内/機微/個人情報）が定義されていること
- 機密区分に応じたマスキング要件が定義されていること
- RAG対象データには検索対象範囲・チャンク方針・メタデータ・根拠提示方法が定義されていること
- RDB/NoSQLに格納するデータが含まれていないこと
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
