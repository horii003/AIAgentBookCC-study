## 2. 各項目の記載ガイド

### 機能ID
- 形式例：`FR-001`
- 後続工程（基本設計・詳細設計・テスト）で不変の識別子

### 機能名
- 機能の内容が一目で分かる簡潔な名称
- 例：  
  - 申請種別判定  
  - 入力情報不足チェック  
  - 申請書自動生成  

### 機能概要
- 機能が「何をするのか」を1〜3行で記載
- **How（実装方法）は書かない**

### 対象業務
- 業務一覧から該当する業務名を記載
- 例：業務処理業務、情報検索業務

### 対象業務プロセス
- 業務プロセス定義のプロセスID／名称
- 例：P-03 申請内容確認

### エージェントの役割
- 人とAIエージェントの分担観点
- 例：
  - 情報収集
  - 判断・提案
  - 自動生成
  - 実行代行

### 自律度
- 自律性・権限レベルを明示
- 例：
  - 参照
  - 提案
  - 実行
  - 実行＋承認依頼

### 入力
- ユーザー入力／他システム入力
- 例：
  - ユーザー自然文
  - 申請条件
  - 業務データ

### 出力
- 機能の結果として生成・提供されるもの
- 例：
  - 判定結果
  - 作成済み申請書
  - 確認メッセージ

### 判断・ルール
- 業務ルール・判断基準との対応
- 例：
  - 金額条件による分岐
  - 役職別承認ルール

### 利用データ
- 業務データ一覧から参照するデータ
- 例：
  - 社員マスタ
  - 申請種別定義

### 連携先／ツール
- 外部システムやエージェントツール
- 例：
  - 申請システムAPI
  - RAG（社内規程）
  - ファイル生成ツール

### 業務要件対応ID
- 以下とのトレーサビリティ確保
  - 要求事項一覧ID
  - 業務要件一覧ID

### 備考
- 補足説明、制約事項、未確定事項
