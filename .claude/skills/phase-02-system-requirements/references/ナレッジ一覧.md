# 役割
あなたはAIエージェント開発のシステム要件定義を専門とする設計者です。

# 指示
業務要件定義とエージェント一覧をもとに、フォーマットに従ったエージェント判断用ナレッジ一覧を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
ナレッジIDは後続工程で不変の識別子として管理すること（KN-xxx）
「ナレッジ＝RAG」ではなく、エージェントの判断に使用するナレッジのみを列挙すること
判断拘束力（強/中/弱）を必ず定義すること
自律度とペアで管理すること（どの自律度レベルまで使用可能か）

## 2. 各項目の記載ガイド

### ナレッジIDの採番ルール
- 形式例：`KN-001`
- 後続工程で不変の識別子

### ナレッジ種別（例）
- 規程・ポリシー（社内規程、法令、ガイドライン）
- 業務ルール（条件分岐、計算ルール、判断基準）
- FAQ／Q&A（運用知見、例外対応）
- マスタ定義（コード定義、分類表）
- 判断事例（過去事例、判断履歴）
- 外部基準（法令、業界標準）
- モデル補助知識（few-shot例、評価基準）

### 判断拘束力
- 強：このナレッジに必ず従う（例：法令・社内規程）
- 中：原則従うが例外あり
- 弱：参考情報（最終判断は他要素）

### 利用自律度
- ナレッジを使用できる最大自律度レベルを明示
- 例：Lv3まで使用可、Lv4以上では禁止

## 3. 受入基準

作成後に以下の受入基準を満たしているか確認すること。
- すべてのナレッジにKN-xxx形式のIDが付与されていること
- エージェントの判断に使用するナレッジのみが列挙されていること（RAGと混同しないこと）
- すべてのナレッジに判断拘束力（強/中/弱）が定義されていること
- 使用可能な最大自律度レベルが明示されていること
- 規程・ポリシー/業務ルール/FAQ・Q&A/マスタ定義のいずれかに分類されていること
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
