## ターゲットユーザー

社内の一般社員（申請ルールや申請方法に詳しくない場合がある）

## 要求事項

- RQ-01: 社員が申請内容を入力すると、必要な申請種別・使用すべき申請書・申請先を提示すること
- RQ-02: 社内の申請ルールを参照して正しい申請種別を判断すること
- RQ-03: 不足情報を対話で確認しながら申請書を自動作成すること
- RQ-04: 申請ミスや差し戻しを減らすこと
- RQ-05: 申請ルールや申請方法が分からない利用者でも正しく申請できること

| BR-03 | 判断した申請種別に対応する申請書と申請先を提示する | RQ-01 | 提案 | High |

## 対象申請種別

本システムが対応する申請種別は以下の2種類のみとする。

| 申請種別 | 申請書名 |
| -------- | -------- |
| 交通費精算申請 | 交通費精算申請書 |
| 経費精算申請 | 経費精算申請書 |

## 申請種別の判断不能ケース

申請内容から交通費精算申請・経費精算申請のいずれかを判断できない場合は、ユーザーに申請種別を確認する。
- ユーザーに「交通費精算申請」「経費精算申請」の選択肢を提示する
- ユーザーの選択に従って対応する申請フローへ誘導する

## 申請種別ごとの申請対象

### 経費精算申請の対象経費区分

| 経費区分 | 具体例 |
| -------- | ------ |
| 事務用品費 | 書籍、文房具、オフィス用品など |
| 宿泊費 | ホテル、宿泊施設など |
| 資格精算費 | 資格試験、受験料、認定費用など |
| その他経費 | 上記以外の業務関連経費 |

### 交通費精算申請の対象交通手段

| 交通手段 | 備考 |
| -------- | ---- |
| 電車 | 経路テーブル検索で運賃を自動計算 |
| バス | 固定運賃 |
| タクシー | 固定運賃 |
| 飛行機 | 固定運賃 |

## 業務種別ごとの業務ルール

### 申請種別判断の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| APL-01 | 申請種別の特定 | 社員の申請内容（目的・金額・行動等）から該当する申請種別（交通費精算申請・経費精算申請）を特定する | High |
| APL-02 | 申請種別の判断不能時の確認 | 申請内容から交通費精算申請・経費精算申請のいずれかを判断できない場合は、ユーザーに「交通費精算申請」「経費精算申請」の選択肢を提示して確認する | High |
| APL-03 | 申請対象外の判断 | 申請内容が交通費精算申請・経費精算申請のいずれにも該当しない場合は、申請不要または担当部署への確認を案内する | Medium |
| APL-04 | 申請先の特定 | 申請種別・申請者の所属部署・金額規模等に応じた申請先（承認者・部署）を提示する | High |

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRP-01 | 移動情報の収集単位 | 移動情報（移動日、出発地、目的地、交通手段、費用、業務目的）を一区間ずつ収集する | High |
| TRP-02 | 交通費の自動計算 | 運賃データから交通費を自動計算する（電車は経路テーブル検索、バス・タクシー・飛行機は固定運賃） | High |
| TRP-03 | 申請期限 | 申請は経費発生日から3ヶ月以内であること | High |
| TRP-04 | 高額事前承認 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | High |
| TRP-05 | 駅名の正規化 | 駅名を正規化する（例：「渋谷駅」→「渋谷」） | High |
| TRP-06 | 業務目的の記載 | すべての申請に業務目的の記載が必要 | High |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXP-01 | 領収書からの情報抽出 | 領収書画像から経費情報（購入日、店舗名、品目、金額）を品目ごとに自動抽出する | High |
| EXP-02 | 経費区分の自動判断 | 品目から経費区分を自動判断する（事務用品費・宿泊費・資格精算費・その他経費の4区分） | High |
| EXP-03 | 申請期限 | 申請は経費発生日から3ヶ月以内であること | High |
| EXP-04 | 高額事前承認 | 経費が5,000円を超える場合は事前に上長の承認が必要 | High |
| EXP-05 | 業務目的の記載 | すべての申請に業務目的の記載が必要 | High |

### 情報収集・対話の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| DLG-01 | 不足情報の確認 | 申請書作成に必要な情報（金額・日程・目的等）が不足している場合は、対話で順次確認する | High |
| DLG-02 | 確認項目の最小化 | 既に入力された情報から推定可能な項目は質問しない | Medium |
| DLG-03 | 入力値の正規化 | 日付・金額・駅名等の入力値は申請書に記載可能な形式に正規化する（例：「来週月曜」→「2026-07-06」、「3万円」→「30,000円」、「渋谷駅」→「渋谷」） | High |

### 申請書自動作成の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| DOC-01 | 申請書の種別対応 | 対象申請種別（交通費精算申請・経費精算申請）ごとに対応する申請書フォームを選択する | High |
| DOC-02 | 申請書の自動入力 | 収集した情報を申請書の各項目に自動入力して申請書を生成する | High |
| DOC-03 | 申請書の確認促進 | 生成した申請書の内容を社員に確認させ、修正が必要な場合は修正できるようにする | High |
