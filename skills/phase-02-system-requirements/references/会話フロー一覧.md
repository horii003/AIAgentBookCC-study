1ステップ＝1ターンを基本とすること
例外分岐を必ず明示すること

## 2. 各項目の記載ガイド

### 会話フロー分類
- 情報収集型（ヒアリング・確認）
- 判断支援型（提案・選択）
- 生成型（文書・データ作成）
- 実行前確認型（承認・最終確認）
- 例外対応型（エラー・差戻し）

### 会話フローIDの採番ルール
- 形式例：`CF-001`
- 後続工程で不変の識別子

### 一覧表の各列の記載方法
- 起点：ユーザー入力／イベント／他会話フロー
- 主エージェント：会話を主導するエージェント（AG-xxx）
- 人の役割：利用者／承認者／確認者など
- 完了条件：「何ができたらこの会話は終わりか」を明確に定義
- 自律度／根拠提示Lv：権限定義・根拠提示方針と一致させる

### 会話ステップ定義（重要）
- 発話内容は「何を言うか」ではなく「何を伝えるか」を記載する
- 判断/処理欄には適用ルール・ナレッジの存在を示す
- 例外分岐は業務要件で想定される例外を必ず明示する
- 根拠提示は根拠提示方針定義と一致させる

### 完了条件
- 「何ができたらこの会話は終わりか」を明確に定義

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- すべての会話フローにCF-xxx形式のIDが付与されていること
- 1ステップ＝1ターンを基本として定義されていること
- 業務要件で想定される例外分岐がすべて明示されていること
- 各ステップの自律度が自律度・権限定義と一致していること
- 各ステップの根拠提示レベルが根拠提示方針と一致していること
- 各会話フローに完了条件が明確に定義されていること
- 情報収集型/判断支援型/生成型/実行前確認型/例外対応型のいずれかに分類されていること
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
