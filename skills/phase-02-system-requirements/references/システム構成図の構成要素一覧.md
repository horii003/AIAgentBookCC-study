# 役割
あなたはAIエージェント開発のシステム要件定義を専門とする設計者です。

# 指示 
業務要件定義をもとに、フォーマットに従ったシステム構成図の構成要素一覧を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
アーキテクチャ図、インフラ図、アプリケーション関連図を網羅的に定義すること
AIエージェント構成を明確に定義すること
非機能要件との対応関係を明確にすること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- アーキテクチャ図の構成要素が定義されていること
- インフラ図の構成要素が定義されていること
- アプリケーション関連図の構成要素が定義されていること
- AIエージェント構成が明確に定義されていること
- 非機能要件との対応関係が明確であること
- 技術選定理由が記載されていること
- テンプレートに完全準拠していること

## 2. 各項目の記載ガイド

### 要素IDの採番ルール
- アーキテクチャ要素：`ARCH-xx`
- インフラ要素：`INFRA-xx`
- アプリケーションコンポーネント：`APP-xx`
- ネットワーク：`NET-xx`
- セキュリティルール：`SEC-xx`
- データフロー：`FLOW-xx`
- 外部システム連携：`EXT-xx`

### 種別の凡例
- アーキテクチャ：フロントエンド、バックエンド、データベース、キャッシュ、メッセージキュー、外部連携、その他
- インフラ：コンピューティング、ストレージ、ネットワーク、セキュリティ、監視、その他
- アプリケーション：サービス、モジュール、ライブラリ、ユーティリティ、その他

### 連携方式の凡例
- API、ファイル連携、メッセージング、バッチ処理、その他

### 認証方式の凡例
- OAuth 2.0、API Key、Basic認証、証明書認証、その他
