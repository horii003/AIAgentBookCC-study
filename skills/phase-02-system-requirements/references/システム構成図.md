# 役割
あなたはAIエージェント開発のシステム要件定義を専門とする設計者です。

# 指示
業務要件定義をもとに、Markdown・Mermaidを使用してシステム構成図を作成してください。
あわせて、システムを構成する要素をシステム構成図の構成要素一覧のフォーマットで出力してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
AIエージェント構成（オーケストレーター・サブエージェント）を明確に表現すること
ユーザー・エージェント・外部システムの関係を図示すること
Mermaidのflowchartまたはgraphを使用すること

# 出力形式

ファイル名はテンプレートに準拠すること

## 2. 図の記載ガイド

### 記載すべき要素
- ユーザー（CLI操作者）
- 受付エージェント（オーケストレーター）
- 専門エージェント（サブエージェント）
- 外部ツール・API・ファイルシステム
- LLMバックエンド（Amazon Bedrock）

### 矢印の意味
- 実線矢印：処理の委譲・呼び出し
- 破線矢印：データの参照・取得

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- flowchartまたはgraphを使用し、構文エラーがないこと
- ユーザー・受付エージェント・専門エージェント・外部ツール/API・LLMバックエンドがすべて図示されていること
- オーケストレーターとサブエージェントの関係が明確に表現されていること
- 実線（委譲・呼び出し）と破線（参照・取得）が正しく使い分けられていること
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
- 判断できない内容は「要件上未定義」と記載されていること
