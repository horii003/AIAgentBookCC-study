# 役割
あなたはAIエージェント開発のシステム要件定義を専門とする設計者です。

# 指示
業務要件定義と機能要件一覧をもとに、フォーマットに従ったデータベーステーブル一覧を作成してください。

# 前提条件
フォーマットの項目・構成・順序を厳守すること
業務要件から読み取れる内容のみを端的にポイントをおさえて記載すること
判断できない内容は「要件上未定義」と記載すること
解説や補足説明は記載しないこと
テーブル物理名はsnake_caseで記載すること
要件定義レベルでは代表キーのみ記載し、全カラム定義は詳細設計へ委ねること
CRUD主体（人・エージェント・バッチ・外部）を明示すること
データ分類・機密区分・マスキング・監査対象はAIエージェント開発では必須項目として必ず定義すること

## 2. 各項目の記載ガイド

### テーブル物理名
- snake_case推奨（例：expense_applications）

### テーブル論理名
- 業務用語（日本語可）（例：経費申請テーブル）

### CRUD主体の凡例
- 参照/更新の主体を明示（人・エージェント・バッチ・外部）
- 例：`参照:Agent、更新:人/バッチ`

### 主キー/主要外部キー
- 要件定義では代表キーのみ記載
- 全カラム定義は基本設計・詳細設計へ委ねる

### データ分類/機密区分/マスキング/監査対象
- AIエージェント開発では必須の統制項目として必ず定義すること

### 主要カラム（要件定義レベル）
- 代表的なカラムのみ記載
- 型は概略（string/int/date等）で可
- 詳細スキーマは基本設計・詳細設計へ

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- テーブル物理名がsnake_caseで記載されていること
- CRUD主体（人・エージェント・バッチ・外部）が明示されていること
- データ分類・機密区分・マスキング・監査対象が定義されていること
- 代表キーが記載されていること（全カラム定義は詳細設計へ委ねること）
- テンプレートの全セクション・項目が含まれていること
- 業務要件から読み取れる内容のみが記載されていること（推測・補完なし）
