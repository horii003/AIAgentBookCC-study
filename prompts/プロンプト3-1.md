# 技術スタック

## 言語とランタイム

- Python 3.x
- エントリーポイント: `main.py`

## コアフレームワーク

- AWS Strands Agents (`strands-agents`, `strands-agents-tools`, `strands-agents-builder`) — マルチエージェント・オーケストレーションフレームワーク Version 1.25.0
- エージェントはLLMバックエンドとしてAmazon Bedrock（Claude Sonnet 4.5）を使用
- モデル設定は `config/model_config.py` に集約

## 主要ライブラリ

- `pydantic` (v2+) — すべての入力と状態モデルのデータバリデーション
- `openpyxl` — 経費申請書のExcelファイル生成
- `boto3` / `botocore` — Bedrockアクセス用AWS SDK
- `Pillow` — 領収書読み取り用の画像処理（`strands_tools` の `image_reader` が内部で使用する間接依存）
- `python-dotenv` — 環境変数管理
- `python-dateutil` — 日付解析

## ナレッジ・外部連携の実現方式

- 申請ルールナレッジ: システムプロンプト埋め込み（RAG不使用）
- 領収書OCR: `strands_tools` の `image_reader` を直接使用

# アーキテクチャ方針

## マルチエージェント構成

- 実装方式: Agent As Tools
- 構成: 階層型マルチエージェント
  - 申請受付窓口エージェント（オーケストレーター）
  - 交通費精算申請エージェント（専門エージェント）
  - 経費精算申請エージェント（専門エージェント）
